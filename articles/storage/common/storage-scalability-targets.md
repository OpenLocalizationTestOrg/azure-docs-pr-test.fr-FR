## <a name="partitions-in-azure-storage"></a><span data-ttu-id="65e92-125">Partitions dans Azure Storage</span><span class="sxs-lookup"><span data-stu-id="65e92-125">Partitions in Azure Storage</span></span>
<span data-ttu-id="65e92-126">Chaque objet contenant des données stockées dans Azure Storage (blobs, messages, entités et fichiers) appartient à une partition et est identifié par une clé de partition.</span><span class="sxs-lookup"><span data-stu-id="65e92-126">Every object that holds data that is stored in Azure Storage (blobs, messages, entities, and files) belongs to a partition, and is identified by a partition key.</span></span> <span data-ttu-id="65e92-127">La partition détermine la manière dont Azure Storage équilibre la charge des blobs, des messages, des entités et des fichiers sur les serveurs pour répondre aux besoins de trafic de ces objets.</span><span class="sxs-lookup"><span data-stu-id="65e92-127">The partition determines how Azure Storage load balances blobs, messages, entities, and files across servers to meet the traffic needs of those objects.</span></span> <span data-ttu-id="65e92-128">La clé de partition est unique. Elle permet de localiser un blob, un message ou une entité.</span><span class="sxs-lookup"><span data-stu-id="65e92-128">The partition key is unique and is used to locate a blob, message, or entity.</span></span>

<span data-ttu-id="65e92-129">Le tableau présenté plus haut dans la section [Objectifs d'évolutivité des comptes de stockage standard](#standard-storage-accounts) répertorie les objectifs de performance pour une seule partition pour chaque service.</span><span class="sxs-lookup"><span data-stu-id="65e92-129">The table shown above in [Scalability Targets for Standard Storage Accounts](#standard-storage-accounts) lists the performance targets for a single partition for each service.</span></span>

<span data-ttu-id="65e92-130">Les partitions affectent l’extensibilité et l’équilibrage de charge de chacun des services de stockage comme suit :</span><span class="sxs-lookup"><span data-stu-id="65e92-130">Partitions affect load balancing and scalability for each of the storage services in the following ways:</span></span>

* <span data-ttu-id="65e92-131">**Blobs** : la clé de partition d’un blob est le nom du compte + le nom du conteneur + le nom du blob.</span><span class="sxs-lookup"><span data-stu-id="65e92-131">**Blobs**: The partition key for a blob is account name + container name + blob name.</span></span> <span data-ttu-id="65e92-132">Cela signifie que chaque objet peut avoir sa propre partition si la charge sur le blob en fait la demande.</span><span class="sxs-lookup"><span data-stu-id="65e92-132">This means that each blob can have its own partition if load on the blob demands it.</span></span> <span data-ttu-id="65e92-133">Les blobs peuvent être répartis sur plusieurs serveurs afin d’offrir un accès horizontal. Toutefois, un blob donné ne peut être servi que par un seul serveur.</span><span class="sxs-lookup"><span data-stu-id="65e92-133">Blobs can be distributed across many servers in order to scale out access to them, but a single blob can only be served by a single server.</span></span> <span data-ttu-id="65e92-134">Si les blobs peuvent être regroupés de manière logique dans des conteneurs, ce regroupement n’a aucune incidence sur le partitionnement.</span><span class="sxs-lookup"><span data-stu-id="65e92-134">While blobs can be logically grouped in blob containers, there are no partitioning implications from this grouping.</span></span>
* <span data-ttu-id="65e92-135">**Fichiers**: la clé de partition pour un fichier est le nom du compte + le nom de partage du fichier.</span><span class="sxs-lookup"><span data-stu-id="65e92-135">**Files**: The partition key for a file is account name + file share name.</span></span> <span data-ttu-id="65e92-136">Cela signifie que tous les fichiers dans un partage de fichiers sont également dans une seule partition.</span><span class="sxs-lookup"><span data-stu-id="65e92-136">This means all files in a file share are also in a single partition.</span></span>
* <span data-ttu-id="65e92-137">**Messages**: la clé de partition d’un message est le nom du compte + le nom de la file d’attente : tous les messages d’une file d’attente sont regroupés en une seule partition et servis par un seul serveur.</span><span class="sxs-lookup"><span data-stu-id="65e92-137">**Messages**: The partition key for a message is the account name + queue name, so all messages in a queue are grouped into a single partition and are served by a single server.</span></span> <span data-ttu-id="65e92-138">Plusieurs files d’attente peuvent être traitées par différents serveurs pour équilibrer la charge, quel que soit le nombre de files d’attente du compte de stockage.</span><span class="sxs-lookup"><span data-stu-id="65e92-138">Different queues may be processed by different servers to balance the load for however many queues a storage account may have.</span></span>
* <span data-ttu-id="65e92-139">**Entités** : la clé de partition d’une entité est le nom du compte + le nom de la table + la clé de partition, cette dernière correspondant à la valeur de la propriété **PartitionKey** requise et définie par l’utilisateur pour l’entité.</span><span class="sxs-lookup"><span data-stu-id="65e92-139">**Entities**: The partition key for an entity is account name + table name + partition key, where the partition key is the value of the required user-defined **PartitionKey** property for the entity.</span></span> <span data-ttu-id="65e92-140">Toutes les entités affichant la même valeur de clé de partition sont regroupées dans la même partition et sont servies sur le même serveur de partition.</span><span class="sxs-lookup"><span data-stu-id="65e92-140">All entities with the same partition key value are grouped into the same partition and are served by the same partition server.</span></span> <span data-ttu-id="65e92-141">Il s’agit d’un point important de la conception de votre application.</span><span class="sxs-lookup"><span data-stu-id="65e92-141">This is an important point to understand in designing your application.</span></span> <span data-ttu-id="65e92-142">Votre application doit offrir un parfait équilibre entre les avantages de l’extensibilité de la répartition des entités sur plusieurs partitions et les avantages d’accès aux données du regroupement des entités en une seule partition.</span><span class="sxs-lookup"><span data-stu-id="65e92-142">Your application should balance the scalability benefits of spreading entities across multiple partitions with the data access advantages of grouping entities in a single partition.</span></span>  

<span data-ttu-id="65e92-143">Grâce au regroupement d’un ensemble d’entités d’une table en une seule partition, il est possible de mener des opérations atomiques par lots sur les entités d’une même partition, car une partition existe sur un serveur unique.</span><span class="sxs-lookup"><span data-stu-id="65e92-143">A key advantage to grouping a set of entities in a table into a single partition is that it's possible to perform atomic batch operations across entities in the same partition, since a partition exists on a single server.</span></span> <span data-ttu-id="65e92-144">Ainsi, si vous souhaitez effectuer des opérations par lot sur un groupe d’entités, pensez à les regrouper avec la même clé de partition.</span><span class="sxs-lookup"><span data-stu-id="65e92-144">Therefore, if you wish to perform batch operations on a group of entities, consider grouping them with the same partition key.</span></span> 

<span data-ttu-id="65e92-145">Par contre, les entités qui se trouvent dans la même table mais qui ont différentes clés de partition peuvent être réparties sur plusieurs serveurs, ce qui permet de disposer d’une meilleure extensibilité.</span><span class="sxs-lookup"><span data-stu-id="65e92-145">On the other hand, entities that are in the same table but have different partition keys can be load balanced across different servers, making it possible to have greater scalability.</span></span>
