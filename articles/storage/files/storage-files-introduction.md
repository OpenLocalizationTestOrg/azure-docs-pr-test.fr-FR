## <a name="why-azure-file-storage-is-useful"></a><span data-ttu-id="4398a-112">Avantages du Stockage Fichier Azure</span><span class="sxs-lookup"><span data-stu-id="4398a-112">Why Azure File storage is useful</span></span>

<span data-ttu-id="4398a-113">Le Stockage Fichier Azure vous permet de remplacer les serveurs de fichiers Windows Server, Linux ou NAS hébergés localement ou sur le cloud grâce à un partage de fichiers sur cloud sans système d’exploitation.</span><span class="sxs-lookup"><span data-stu-id="4398a-113">Azure File storage allows you to replace Windows Server, Linux, or NAS-based file servers hosted on-premises or in the cloud with an OS-free cloud file share.</span></span> <span data-ttu-id="4398a-114">Le Stockage Fichier Azure possède les avantages suivants :</span><span class="sxs-lookup"><span data-stu-id="4398a-114">Azure File storage has the following benefits:</span></span>

* <span data-ttu-id="4398a-115">**Un accès partagé** : les partages de fichiers Azure prennent en charge les protocoles standards SMB. Cela signifie que vous pouvez facilement remplacer vos partages de fichiers locaux par des partages de fichiers Azure, sans vous soucier de la compatibilité des applications.</span><span class="sxs-lookup"><span data-stu-id="4398a-115">**Shared access** Azure File shares support the industry standard SMB protocol, meaning you can seamlessly replace your on-premises file shares with Azure File shares without worrying about application compatibility.</span></span> <span data-ttu-id="4398a-116">Être en mesure d’accéder à un partage de fichier sur plusieurs machines et applications / instances peut s’avérer très avantageux avec le Stockage Fichier Azure.</span><span class="sxs-lookup"><span data-stu-id="4398a-116">Being able to access a file share from multiple machines and applications/instances is a significant advantage with Azure File storage.</span></span>

* <span data-ttu-id="4398a-117">**Entièrement gérés** : les partages de fichiers Azure peuvent être créés sans avoir à gérer le matériel ou un système d’exploitation. Vous n’êtes donc pas obligé d’appliquer au serveur du système d’exploitation des mises à jour de sécurité critiques, ni de remplacer les disques durs défaillants.</span><span class="sxs-lookup"><span data-stu-id="4398a-117">**Fully Managed** Azure File shares can be created without the need to manage hardware or an OS, which means you don't have to deal with patching the server OS with critical security upgrades or replacing faulty hard disks.</span></span>

* <span data-ttu-id="4398a-118">**Scripts et outils** : les cmdlets PowerShell et Azure CLI peuvent créer, monter et gérer les partages de fichier Azure dans le cadre de l’administration des applications Azure.</span><span class="sxs-lookup"><span data-stu-id="4398a-118">**Scripting and Tooling** PowerShell cmdlets and Azure CLI can be used to create, mount, and manage Azure File shares as part of the administration of Azure applications.</span></span> <span data-ttu-id="4398a-119">Vous pouvez créer et gérer les partages de fichiers Azure à l’aide du [portail Azure](https://portal.azure.com) et de l’[Explorateur de stockage Azure](https://storageexplorer.com).</span><span class="sxs-lookup"><span data-stu-id="4398a-119">You can create and manage Azure File shares using the [Azure portal](https://portal.azure.com) and the [Azure Storage Explorer](https://storageexplorer.com).</span></span> 

* <span data-ttu-id="4398a-120">**Résilience** : le Stockage Fichier Azure a été entièrement conçu de manière à être toujours disponible.</span><span class="sxs-lookup"><span data-stu-id="4398a-120">**Resiliency** Azure File storage has been built from the ground up to be always available.</span></span> <span data-ttu-id="4398a-121">L’utilisation du Stockage Fichier Azure au lieu des partages de fichiers locaux signifie que vous n’avez plus à vous soucier des pannes de courant ou des problèmes de réseau.</span><span class="sxs-lookup"><span data-stu-id="4398a-121">Replacing on-premises file shares with Azure File storage means you no longer have to wake up to deal with local power outages or network issues.</span></span> 

* <span data-ttu-id="4398a-122">**Programmabilité familière** : les applications exécutées dans Azure peuvent accéder aux données dans le partage via les [API d’E/S du système de fichier](https://msdn.microsoft.com/library/system.io.file.aspx).</span><span class="sxs-lookup"><span data-stu-id="4398a-122">**Familiar Programmability** Applications running in Azure can access data on the share via [file system I/O APIs](https://msdn.microsoft.com/library/system.io.file.aspx).</span></span> <span data-ttu-id="4398a-123">Les développeurs peuvent ainsi tirer profit de leur code et compétences actuels pour migrer les applications existantes.</span><span class="sxs-lookup"><span data-stu-id="4398a-123">Developers can therefore leverage their existing code and skills to migrate existing applications.</span></span> <span data-ttu-id="4398a-124">En plus des API d’E/S du système, vous pouvez utiliser n’importe quelle bibliothèque de client du stockage Azure, comme c’est le cas pour [.NET](/dotnet/api/overview/azure/storage?view=azure-dotnet) ou l’[API REST du stockage Azure](/rest/api/storageservices/file-service-rest-api).</span><span class="sxs-lookup"><span data-stu-id="4398a-124">In addition to System IO APIs, you can use any of the Azure storage client Libraries, such as the one for [.NET](/dotnet/api/overview/azure/storage?view=azure-dotnet), or the [Azure Storage REST API](/rest/api/storageservices/file-service-rest-api).</span></span>

<span data-ttu-id="4398a-125">Les partages de fichiers Azure peuvent être utilisés pour :</span><span class="sxs-lookup"><span data-stu-id="4398a-125">Azure File shares can be used to:</span></span>

* <span data-ttu-id="4398a-126">**Remplacer des serveurs de fichiers locaux** : le Stockage Fichier Azure peut être utilisé pour remplacer complètement les partages de fichiers sur les serveurs de fichiers locaux traditionnels ou les périphériques NAS.</span><span class="sxs-lookup"><span data-stu-id="4398a-126">**Replace on-premises file servers** Azure File storage can be used to completely replace file shares on traditional on-premises file servers or NAS devices.</span></span> <span data-ttu-id="4398a-127">Les systèmes d’exploitation courants tels que Windows, Mac OS et Linux facilement peuvent facilement un partage de fichiers Azure, n’importe où dans le monde.</span><span class="sxs-lookup"><span data-stu-id="4398a-127">Popular operating systems such as Windows, macOS, and Linux can easily mount an Azure File share wherever they are in the world.</span></span>

    <span data-ttu-id="4398a-129">Le Stockage Fichier Azure permet d’effectuer facilement la migration « lift-and-shift » vers le cloud des applications qui utilisent des partages de fichiers locaux pour partager des données entre plusieurs parties d’une application.</span><span class="sxs-lookup"><span data-stu-id="4398a-129">Azure File storage makes it easy to "lift and shift" applications to the cloud that use on-premises file shares to share data between parts of the application.</span></span> <span data-ttu-id="4398a-130">Pour implémenter cela, chaque machine virtuelle se connecte au partage de fichiers, après quoi elle peut lire et écrire dans les fichiers comme sur un partage de fichiers local.</span><span class="sxs-lookup"><span data-stu-id="4398a-130">To implement this, each VM connects to the file share and then it can read and write files just like it would against an on-premises file share.</span></span>

    * <span data-ttu-id="4398a-133">**Paramètres d’application partagés**</span><span class="sxs-lookup"><span data-stu-id="4398a-133">**Shared Application Settings**</span></span>
