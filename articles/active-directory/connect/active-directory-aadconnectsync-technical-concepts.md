## <a name="connector"></a><span data-ttu-id="e2349-115">Connecteur</span><span class="sxs-lookup"><span data-stu-id="e2349-115">Connector</span></span>

<span data-ttu-id="e2349-117">Ils sont installés sur l’ordinateur exécutant Azure AD Connect Sync.</span><span class="sxs-lookup"><span data-stu-id="e2349-117">These are installed on the computer running Azure AD Connect sync.</span></span>

<span data-ttu-id="e2349-122">Les importations et exportations ont lieu uniquement quand elles sont planifiées, ce qui offre une isolation supplémentaire par rapport aux modifications qui se produisent dans le système, dans la mesure où les modifications ne se propagent pas automatiquement à la source de données connectée.</span><span class="sxs-lookup"><span data-stu-id="e2349-122">Imports and exports only occur when scheduled, allowing for further insulation from changes occurring within the system, since changes do not automatically propagate to the connected data source.</span></span> <span data-ttu-id="e2349-123">En outre, les développeurs peuvent également créer leurs propres connecteurs pour se connecter à pratiquement n'importe quelle source de données.</span><span class="sxs-lookup"><span data-stu-id="e2349-123">In addition, developers may also create their own connectors for connecting to virtually any data source.</span></span>

## <a name="connector-space"></a><span data-ttu-id="e2349-134">Espace de connecteur</span><span class="sxs-lookup"><span data-stu-id="e2349-134">Connector space</span></span>

<span data-ttu-id="e2349-136">Cela permet au service de synchronisation de s’exécuter localement sans qu’il soit nécessaire de contacter le système distant lors de la synchronisation des objets et cela limite l’interaction aux importations et exportations.</span><span class="sxs-lookup"><span data-stu-id="e2349-136">This allows the sync service to operate locally without the need to contact the remote system when synchronizing the objects and restricts interaction to imports and exports only.</span></span>

<span data-ttu-id="e2349-137">Quand la source de données et le connecteur peuvent fournir une liste de modifications (une importation delta), l’efficacité opérationnelle augmente considérablement car seules les modifications apportées depuis le dernier cycle d’interrogation sont échangées.</span><span class="sxs-lookup"><span data-stu-id="e2349-137">When the data source and the connector have the ability to provide a list of changes (a delta import), then the operational efficiency increases dramatically as only changes since the last polling cycle are exchanged.</span></span> <span data-ttu-id="e2349-138">L’espace de connecteur isole la source de données connectée des modifications qui se propagent automatiquement en exigeant que le connecteur planifie les importations et les exportations.</span><span class="sxs-lookup"><span data-stu-id="e2349-138">The connector space insulates the connected data source from changes propagating automatically by requiring that the connector schedule imports and exports.</span></span> <span data-ttu-id="e2349-139">Cette assurance supplémentaire vous procure une tranquillité d’esprit lors des tests, de l’examen ou de la confirmation de la mise à jour suivante.</span><span class="sxs-lookup"><span data-stu-id="e2349-139">This added insurance grants you peace of mind while testing, previewing, or confirming the next update.</span></span>

## <a name="metaverse"></a><span data-ttu-id="e2349-140">Métaverse</span><span class="sxs-lookup"><span data-stu-id="e2349-140">Metaverse</span></span>
<span data-ttu-id="e2349-141">Le métaverse est l’affichage consolidé de toutes les identités jointes des espaces de connecteur voisins.</span><span class="sxs-lookup"><span data-stu-id="e2349-141">The metaverse is the consolidated view of all joined identities from neighboring connector spaces.</span></span>

<span data-ttu-id="e2349-142">À mesure que des identités sont liées et que l’autorité est attribuée pour différents attributs via des mappages de flux d’importation, l’objet de métaverse central commence à regrouper les informations provenant de plusieurs systèmes.</span><span class="sxs-lookup"><span data-stu-id="e2349-142">As identities are linked together and authority is assigned for various attributes through import flow mappings, the central metaverse object begins to aggregate information from multiple systems.</span></span> <span data-ttu-id="e2349-143">À partir de ce flux d’attributs d’objets, des mappages transmettent des informations aux systèmes sortants.</span><span class="sxs-lookup"><span data-stu-id="e2349-143">From this object attribute flow, mappings carry information to outbound systems.</span></span>

<span data-ttu-id="e2349-144">Des objets sont créés quand un système faisant autorité les projette dans le métaverse.</span><span class="sxs-lookup"><span data-stu-id="e2349-144">Objects are created when an authoritative system projects them into the metaverse.</span></span> <span data-ttu-id="e2349-145">Dès que toutes les connexions sont supprimées, l’objet de métaverse est supprimé.</span><span class="sxs-lookup"><span data-stu-id="e2349-145">As soon as all connections are removed, the metaverse object is deleted.</span></span>

<span data-ttu-id="e2349-146">Impossible de modifier directement les objets de métaverse.</span><span class="sxs-lookup"><span data-stu-id="e2349-146">Objects in the metaverse cannot be edited directly.</span></span> <span data-ttu-id="e2349-147">Toutes les données de l'objet doivent être fournies via le flux des attributs.</span><span class="sxs-lookup"><span data-stu-id="e2349-147">All data in the object must be contributed through attribute flow.</span></span> <span data-ttu-id="e2349-148">Le métaverse conserve les connecteurs permanents avec chaque espace de connecteur.</span><span class="sxs-lookup"><span data-stu-id="e2349-148">The metaverse maintains persistent connectors with each connector space.</span></span> <span data-ttu-id="e2349-149">Ces connecteurs ne nécessitent pas de réévaluation pour chaque exécution de la synchronisation.</span><span class="sxs-lookup"><span data-stu-id="e2349-149">These connectors do not require reevaluation for each synchronization run.</span></span> <span data-ttu-id="e2349-150">Cela signifie qu'Azure AD Connect Sync n'a pas à localiser à chaque fois l'objet distant correspondant.</span><span class="sxs-lookup"><span data-stu-id="e2349-150">This means that Azure AD Connect sync does not have to locate the matching remote object each time.</span></span> <span data-ttu-id="e2349-151">On n'a donc pas besoin d'agents onéreux pour empêcher des modifications des attributs qui seraient normalement responsables de la mise en corrélation des objets.</span><span class="sxs-lookup"><span data-stu-id="e2349-151">This avoids the need for costly agents to prevent changes to attributes that would normally be responsible for correlating the objects.</span></span>

<span data-ttu-id="e2349-152">Lors de la découverte de nouvelles sources de données pouvant contenir des objets à gérer, Azure AD Connect Sync utilise un processus appelé règle de jointure pour évaluer les candidats potentiels avec lesquels établir un lien.</span><span class="sxs-lookup"><span data-stu-id="e2349-152">When discovering new data sources that may have preexisting objects that need to be managed, Azure AD Connect sync uses a process called a join rule to evaluate potential candidates with which to establish a link.</span></span>
<span data-ttu-id="e2349-153">Une fois le lien établi, cette évaluation ne se reproduit pas et le flux d’attributs normal peut se produire entre la source de données connectée et le métaverse.</span><span class="sxs-lookup"><span data-stu-id="e2349-153">Once the link is established, this evaluation does not reoccur and normal attribute flow can occur between the remote connected data source and the metaverse.</span></span>

## <a name="provisioning"></a><span data-ttu-id="e2349-154">Approvisionnement</span><span class="sxs-lookup"><span data-stu-id="e2349-154">Provisioning</span></span>
<span data-ttu-id="e2349-155">Quand une source faisant autorité projette un nouvel objet dans le métaverse, un nouvel objet d’espace de connecteur peut être créé dans un autre connecteur représentant une source de données connectée en aval.</span><span class="sxs-lookup"><span data-stu-id="e2349-155">When an authoritative source projects a new object into the metaverse a new connector space object can be created in another Connector representing a downstream connected data source.</span></span>

<span data-ttu-id="e2349-156">Cela établit intrinsèquement un lien, et le flux d’attributs peut se produire de manière bidirectionnelle.</span><span class="sxs-lookup"><span data-stu-id="e2349-156">This inherently establishes a link, and attribute flow can proceed bi-directionally.</span></span>

<span data-ttu-id="e2349-157">Chaque fois qu’une règle détermine qu’un nouvel objet d’espace de connecteur doit être créé, on emploie le terme d’« approvisionnement ».</span><span class="sxs-lookup"><span data-stu-id="e2349-157">Whenever a rule determines that a new connector space object needs to be created, it is called provisioning.</span></span> <span data-ttu-id="e2349-158">Toutefois, étant donné que cette opération n’a lieu que dans l’espace de connecteur, elle n’est reportée dans la source de données connectée qu’une fois qu’une exportation est effectuée.</span><span class="sxs-lookup"><span data-stu-id="e2349-158">However, because this operation only takes place within the connector space, it does not carry over into the connected data source until an export is performed.</span></span>
