## <a name="how-it-works"></a><span data-ttu-id="30896-113">Fonctionnement</span><span class="sxs-lookup"><span data-stu-id="30896-113">How it works</span></span>
![Présentation du CDN](./media/cdn-overview/cdn-overview.png)

1. <span data-ttu-id="30896-115">Un utilisateur (Alice) demande un fichier (également appelé ressource) à l’aide d’une URL avec un nom de domaine spécial, par exemple `<endpointname>.azureedge.net`.</span><span class="sxs-lookup"><span data-stu-id="30896-115">A user (Alice) requests a file (also called an asset) using a URL with a special domain name, such as `<endpointname>.azureedge.net`.</span></span>  <span data-ttu-id="30896-116">Le DNS achemine la demande à l’emplacement du point de présence (POP) le plus performant.</span><span class="sxs-lookup"><span data-stu-id="30896-116">DNS routes the request to the best performing Point-of-Presence (POP) location.</span></span>  <span data-ttu-id="30896-117">Il s’agit en général du point de présence le plus proche géographiquement de l'utilisateur.</span><span class="sxs-lookup"><span data-stu-id="30896-117">Usually this is the POP that is geographically closest to the user.</span></span>
2. <span data-ttu-id="30896-118">Si les serveurs Edge du point de présence ne disposent pas du fichier dans leur cache, le serveur Edge demande le fichier à l'origine.</span><span class="sxs-lookup"><span data-stu-id="30896-118">If the edge servers in the POP do not have the file in their cache, the edge server requests the file from the origin.</span></span>  <span data-ttu-id="30896-119">L'origine peut être une application web Azure, un service cloud Azure, un compte de stockage Azure ou n'importe quel serveur web accessible publiquement.</span><span class="sxs-lookup"><span data-stu-id="30896-119">The origin can be an Azure Web App, Azure Cloud Service, Azure Storage account, or any publicly accessible web server.</span></span>
3. <span data-ttu-id="30896-120">L'origine renvoie les fichiers sur le serveur Edge, notamment les en-têtes HTTP facultatifs décrivant la durée de vie du fichier.</span><span class="sxs-lookup"><span data-stu-id="30896-120">The origin returns the file to the edge server, including optional HTTP headers describing the file's Time-to-Live (TTL).</span></span>
4. <span data-ttu-id="30896-121">Le serveur Edge met en cache le fichier et le renvoie au demandeur d'origine (Alice).</span><span class="sxs-lookup"><span data-stu-id="30896-121">The edge server caches the file and returns the file to the original requestor (Alice).</span></span>  <span data-ttu-id="30896-122">Le fichier reste en cache sur le serveur Edge jusqu’à la fin de la durée de vie.</span><span class="sxs-lookup"><span data-stu-id="30896-122">The file remains cached on the edge server until the TTL expires.</span></span>  <span data-ttu-id="30896-123">Si l’origine n’a pas spécifié de durée de vie, elle est par défaut de 7 jours.</span><span class="sxs-lookup"><span data-stu-id="30896-123">If the origin didn't specify a TTL, the default TTL is seven days.</span></span>
5. <span data-ttu-id="30896-124">Des utilisateurs supplémentaires peuvent demander le même fichier à l’aide de la même URL et peuvent également être dirigés vers ce même point de présence.</span><span class="sxs-lookup"><span data-stu-id="30896-124">Additional users may then request the same file using that same URL, and may also be directed to that same POP.</span></span>
6. <span data-ttu-id="30896-125">Si la durée de vie du fichier n'a pas expiré, le serveur Edge renvoie le fichier à partir du cache.</span><span class="sxs-lookup"><span data-stu-id="30896-125">If the TTL for the file hasn't expired, the edge server returns the file from the cache.</span></span>  <span data-ttu-id="30896-126">L’expérience utilisateur est en conséquence plus rapide et plus réactive.</span><span class="sxs-lookup"><span data-stu-id="30896-126">This results in a faster, more responsive user experience.</span></span>
