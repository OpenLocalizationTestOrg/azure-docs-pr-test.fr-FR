## <a name="sessions-and-activities"></a><span data-ttu-id="cfaae-113">Sessions et activités</span><span class="sxs-lookup"><span data-stu-id="cfaae-113">Sessions and activities</span></span>

<span data-ttu-id="cfaae-117">Une activité est identifiée par un nom (limité à 64 caractères) et peut éventuellement incorporer des données supplémentaires (dans la limite de 1 024 octets).</span><span class="sxs-lookup"><span data-stu-id="cfaae-117">An activity is identified by a name (limited to 64 characters) and can optionally embed some extra data (in the limit of 1024 bytes).</span></span>

## <a name="application-information"></a><span data-ttu-id="cfaae-137">Informations de l'application</span><span class="sxs-lookup"><span data-stu-id="cfaae-137">Application information</span></span>
<span data-ttu-id="cfaae-138">Les informations de l’application sont utilisées pour baliser les utilisateurs, c’est-à-dire pour associer des données aux utilisateurs d’une application (elles sont assimilables aux cookies web, sauf qu’elles sont stockées côté serveur sur la plateforme Azure Mobile Engagement).</span><span class="sxs-lookup"><span data-stu-id="cfaae-138">Application information (or app info) is used to tag users, that is, to associate some data to the users of an application (this is similar to web cookies, except that app info is stored on the server side on the Azure Mobile Engagement platform).</span></span>

<span data-ttu-id="cfaae-139">Une information de l’application peut être enregistrée à l’aide de l’API du SDK Mobile Engagement ou de l’API d’appareil de la plateforme Mobile Engagement.</span><span class="sxs-lookup"><span data-stu-id="cfaae-139">App info can be registered by using the Mobile Engagement SDK API or by using the Mobile Engagement platform Device API.</span></span>

<span data-ttu-id="cfaae-140">Une information de l’application est une paire clé/valeur associée à un appareil.</span><span class="sxs-lookup"><span data-stu-id="cfaae-140">App info is a key/value pair associated to a device.</span></span> <span data-ttu-id="cfaae-141">La clé est le nom de l'information de l'application (limité à 64 lettres ASCII [a-zA-Z], chiffres [0-9] et traits de soulignement [_]).</span><span class="sxs-lookup"><span data-stu-id="cfaae-141">The key is the name of the app info (limited to 64 ASCII letters [a-zA-Z], numbers [0-9] and underscores [_]).</span></span> <span data-ttu-id="cfaae-142">La valeur (limitée à 1 024 caractères) peut être une chaîne, un entier, une date (aaaa-MM-jj) ou une valeur booléenne (true ou false).</span><span class="sxs-lookup"><span data-stu-id="cfaae-142">The value (limited to 1024 characters) can be any string, integer, date (yyyy-MM-dd) or Boolean (true or false).</span></span>

<span data-ttu-id="cfaae-143">N'importe quelle quantité d'informations de l'application peut être associée à un appareil, dans les limites définies par les conditions de tarification de Mobile Engagement.</span><span class="sxs-lookup"><span data-stu-id="cfaae-143">Any number of app info can be associated to a device, within the limits defined by the Mobile Engagement pricing terms.</span></span> <span data-ttu-id="cfaae-144">Pour une clé donnée, Mobile Engagement effectue le suivi uniquement de la dernière valeur définie (aucun historique).</span><span class="sxs-lookup"><span data-stu-id="cfaae-144">For one given key, Mobile Engagement only keeps track of the latest value set (no history).</span></span> <span data-ttu-id="cfaae-145">La définition ou la modification de la valeur d'une information de l'application oblige Mobile Engagement à réévaluer les critères d'audience définis sur cette information de l'application (le cas échéant), ce qui signifie que les informations de l'application peuvent être utilisées pour déclencher des opérations push en temps réel.</span><span class="sxs-lookup"><span data-stu-id="cfaae-145">Setting or changing the value of an app info forces Mobile Engagement to re-evaluate audience criteria set on this app info (if any) meaning that app info can be used to trigger realtime pushes.</span></span>

## <a name="extra-data"></a><span data-ttu-id="cfaae-146">Données supplémentaires</span><span class="sxs-lookup"><span data-stu-id="cfaae-146">Extra data</span></span>
<span data-ttu-id="cfaae-147">Les données supplémentaires (ou extras) sont des données arbitraires qui peuvent être associées à des événements, des erreurs, des activités et des tâches.</span><span class="sxs-lookup"><span data-stu-id="cfaae-147">Extra data (or extras) is some arbitrary data that can be attached to events, errors, activities and jobs.</span></span>

<span data-ttu-id="cfaae-148">Les extras sont structurés de la même façon que les objets JSON : ils sont constitués d'une arborescence de paires clé/valeur.</span><span class="sxs-lookup"><span data-stu-id="cfaae-148">Extras are structured similarly to JSON objects: they are made of a tree of key/value pairs.</span></span> <span data-ttu-id="cfaae-149">Les clés sont limitées à 64 lettres ASCII [a-zA-Z], chiffres [0-9] et traits de soulignement [_]) et la taille totale des extras est limitée à 1 024 caractères (une fois encodés au format JSON par le SDK Mobile Engagement).</span><span class="sxs-lookup"><span data-stu-id="cfaae-149">Keys are limited to 64 ASCII letters [a-zA-Z], numbers [0-9] and underscores [_]) and the total size of extras is limited to 1024 characters (once encoded in JSON by the Mobile Engagement SDK).</span></span>

<span data-ttu-id="cfaae-150">L'intégralité de l'arborescence de paires clé/valeur est stockée en tant qu'objet JSON.</span><span class="sxs-lookup"><span data-stu-id="cfaae-150">The whole tree of key/value pairs is stored as a JSON object.</span></span> <span data-ttu-id="cfaae-151">Toutefois, seul le premier niveau de clés/valeurs est décomposé pour être directement accessible à certaines fonctions avancées, telles que les segments (par exemple, vous pouvez facilement définir un segment appelé « Fans SciFi » constitué de tous les utilisateurs ayant envoyé au moins 10 fois l’événement « content_viewed » avec la clé extra « content_type » défini sur la valeur « scifi » le mois dernier).</span><span class="sxs-lookup"><span data-stu-id="cfaae-151">Nevertheless, only the first level of keys/values is decomposed to be directly accessible to some advanced functions like Segments (for example, you can easily define a segment called “SciFi fans” that is made of all users having sent at least 10 times the event named “content_viewed” with the extra key “content_type” set to the value “scifi” in the last month).</span></span> <span data-ttu-id="cfaae-152">Il est donc fortement recommandé de n’envoyer que des extras constitués de listes simples de paires clé/valeur utilisant des valeurs scalaires (par exemple, chaînes, dates, entiers ou valeurs booléennes).</span><span class="sxs-lookup"><span data-stu-id="cfaae-152">It is thus highly recommended to send only extras made of simple lists of key/value pairs using scalar values (for example, strings, dates, integers or Boolean).</span></span>
