# <a name="how-to-log-a-security-event-support-ticket"></a>Enregistrer un ticket de support d’événement de sécurité

1. Accédez à [Support de l’éditeur](https://support.microsoft.com/en-us/getsupport?wf=0&tenant=ClassicCommercial&oaspworkflow=start_1.0.0.0&locale=en-us&supportregion=en-us&pesid=16230&ccsid=636450758943226673) et connectez-vous avec vos informations d’identification Microsoft.
2. Sélectionnez « Événement de sécurité » comme type de problème et choisissez entre les catégories « Incident de sécurité » et « Vulnérabilité ».

    ![Type d’événement et définition][1]

3. Après avoir sélectionné le type de problème et la catégorie, cliquez sur le bouton **Début de la requête**. Fournissez les informations suivantes pour nous aider à mieux comprendre le problème.

    i. Quel est le problème et/ou la vulnérabilité ?

    ii. Pour les vulnérabilités, indiquez le CVE (mitre.org) ou remplissez la calculatrice CVSS3 v3 (https://www.first.org/cvss/calculator/3.0).

    iii. Existe-t-il une résolution ou atténuation ? Si oui, indiquez les étapes de correction.

    iv. Voulez-vous adresser un message à vos clients ? Nous travaillons avec vous pour rédiger un message approprié, le cas échéant.

4. Confirmation de la soumission - Une fois votre demande soumise, nous accuserons réception dans un délai d’un jour ouvrable et affecterons à votre problème une priorité et une gravité.

    - Si vous avez besoin de communiquer avec nous sur votre problème, utilisez le numéro de confirmation dans toute correspondance.
    - Vous pouvez vous informer de la progression de votre demande à tout moment.

5. Étapes suivantes En fonction du problème et de sa gravité, les étapes suivantes peuvent être effectuées :

    - Nous vous communiquerons la conclusion de notre évaluation. En fonction de la conclusion, nous pouvons supprimer ou demander que vous modifiiez votre offre. Dans ce cas, nous travaillons avec vous pour garantir une interruption minime pour les clients concernés.
    - Nous travaillons avec vous pour atténuer l’impact de l’incident/la vulnérabilité pour nos clients communs.
