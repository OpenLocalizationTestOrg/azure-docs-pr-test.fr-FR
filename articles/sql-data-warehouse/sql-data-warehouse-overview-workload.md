## <a name="new-to-data-warehousing"></a><span data-ttu-id="68124-108">Vous découvrez l’entrepôt de données ?</span><span class="sxs-lookup"><span data-stu-id="68124-108">New to data warehousing?</span></span>
<span data-ttu-id="68124-109">Un entrepôt de données est une collection de données qui est chargée d’une ou plusieurs sources de données et qui est utilisée pour l’exécution de tâches décisionnelles, comme la génération de rapports ou l’analyse de données.</span><span class="sxs-lookup"><span data-stu-id="68124-109">A data warehouse is a collection of data that is loaded from one or more data sources and is used to perform business intelligence tasks such as reporting and data analysis.</span></span>

<span data-ttu-id="68124-110">Les entrepôts de données incluent des requêtes qui analysent un grand nombre de lignes, de grands volumes de données et qui peuvent renvoyer des résultats relativement volumineux à des fins d’analyse et de génération de rapports.</span><span class="sxs-lookup"><span data-stu-id="68124-110">Data warehouses are characterized by queries that scan larger numbers of rows, large ranges of data and may return relatively large results for the purposes of analysis and reporting.</span></span> <span data-ttu-id="68124-111">Les entrepôts de données sont également caractérisés par des charges de données relativement importantes, alors que les opérations d’insertion/de mise à jour/de suppression au niveau transactionnel sont de petite taille.</span><span class="sxs-lookup"><span data-stu-id="68124-111">Data warehouses are also characterized by relatively large data loads versus small transaction-level inserts/updates/deletes.</span></span>

* <span data-ttu-id="68124-112">Un entrepôt de données est davantage performant lorsque la configuration de stockage des données est idéale pour les requêtes qui doivent analyser un grand nombre de lignes ou de gros volumes de données.</span><span class="sxs-lookup"><span data-stu-id="68124-112">A data warehouse performs best when the data is stored in a way that optimizes queries that need to scan large numbers of rows or large ranges of data.</span></span> <span data-ttu-id="68124-113">Ce type de tâches d’analyse est plus efficace lorsque les données sont stockées et classées par colonnes, non en lignes.</span><span class="sxs-lookup"><span data-stu-id="68124-113">This type of scanning works best when the data is stored and searched by columns, instead of by rows.</span></span>

> [!NOTE]
> <span data-ttu-id="68124-114">L’index Columnstore en mémoire, qui utilise le stockage par colonne, procure une compression jusqu'à 10 fois supérieure et des requêtes jusqu'à 100 fois supérieures par rapport aux arborescences binaires traditionnelles dédiées aux requêtes de rapports et d'analyse.</span><span class="sxs-lookup"><span data-stu-id="68124-114">The in-memory columnstore index, which uses column storage, provides up to 10x compression gains and 100x query performance gains over traditional binary trees for reporting and analytics queries.</span></span> <span data-ttu-id="68124-115">Nous considérons les index Columnstore comme la norme en matière de stockage et d’analyse de gros volumes de données dans un entrepôt de données.</span><span class="sxs-lookup"><span data-stu-id="68124-115">We consider columnstore indexes as the standard for storing and scanning large data in a data warehouse.</span></span>
>

* <span data-ttu-id="68124-116">Un entrepôt de données présente des exigences différentes d’un système ; il est configuré pour le traitement transactionnel en ligne (OLTP, online transaction processing).</span><span class="sxs-lookup"><span data-stu-id="68124-116">A data warehouse has different requirements than a system that optimizes for online transaction processing (OLTP).</span></span> <span data-ttu-id="68124-117">Le système OLTP propose de multiples opérations d’insertion, de mise à jour et de suppression.</span><span class="sxs-lookup"><span data-stu-id="68124-117">The OLTP system has many insert, update, and delete operations.</span></span> <span data-ttu-id="68124-118">Ces opérations sont dirigées vers des lignes spécifiques de la table.</span><span class="sxs-lookup"><span data-stu-id="68124-118">These operations seek to specific rows in the table.</span></span> <span data-ttu-id="68124-119">Les recherches de table aboutissent à de meilleurs résultats lorsque les données sont stockées par lignes.</span><span class="sxs-lookup"><span data-stu-id="68124-119">Table seeks perform best when the data is stored in a row-by-row manner.</span></span> <span data-ttu-id="68124-120">Les données peuvent être triées et recherchées rapidement à l’aide d’une approche de type « Diviser pour régner », appelée recherche par arborescence binaire.</span><span class="sxs-lookup"><span data-stu-id="68124-120">The data can be sorted and quickly searched with a divide and conquer approach called a binary tree or btree search.</span></span>

<span data-ttu-id="68124-122">Le chargement des données est une phase importante de la charge de travail de l’entrepôt de données.</span><span class="sxs-lookup"><span data-stu-id="68124-122">Data loading is a big part of the data warehouse workload.</span></span> <span data-ttu-id="68124-123">Les entreprises sont généralement équipées d’un système OLTP chargé qui effectue le suivi des modifications tout au long de la journée, à mesure que les clients génèrent des transactions commerciales.</span><span class="sxs-lookup"><span data-stu-id="68124-123">Businesses usually have a busy OLTP system that tracks changes throughout the day when customers are generating business transactions.</span></span> <span data-ttu-id="68124-124">Régulièrement, souvent le soir durant une période de maintenance, les transactions sont déplacées ou copiées sur l’entrepôt de données.</span><span class="sxs-lookup"><span data-stu-id="68124-124">Periodically, often at night during a maintenance window, the transactions are moved or copied to the data warehouse.</span></span> <span data-ttu-id="68124-125">Une fois que les données sont dans l’entrepôt de données, les analystes peuvent procéder à l’analyse et prendre des décisions métiers relatives aux données.</span><span class="sxs-lookup"><span data-stu-id="68124-125">Once the data is in the data warehouse, analysts can perform analysis and make business decisions on the data.</span></span>

<span data-ttu-id="68124-134">Les requêtes de rapports et d’analyse sont bien souvent classées en tant que requêtes réduites, intermédiaires ou importantes en fonction du nombre de critères, mais elles sont généralement basées sur des intervalles de temps.</span><span class="sxs-lookup"><span data-stu-id="68124-134">Reporting and analysis queries are often categorized into small, medium and large based on a number of criteria, but usually it is based on time.</span></span> <span data-ttu-id="68124-135">La plupart des entrepôts de données comportent une charge de travail hybride, comprenant à la fois des requêtes à courte échéance et à longue échéance.</span><span class="sxs-lookup"><span data-stu-id="68124-135">In most data warehouses, there is a mixed workload of fast-running versus long-running queries.</span></span> <span data-ttu-id="68124-136">Pour chaque cas, il est important d’identifier ce mélange et de déterminer sa fréquence (horaire, quotidienne, fin du mois, fin de trimestre, etc.).</span><span class="sxs-lookup"><span data-stu-id="68124-136">In each case, it is important to determine this mix and to determine its frequency (hourly, daily, month-end, quarter-end, and so on).</span></span> <span data-ttu-id="68124-137">Il est essentiel de comprendre que cette charge de travail hybride de requêtes, couplée à l’accès concurrentiel, permet de planifier de manière appropriée la capacité d’un entrepôt de données.</span><span class="sxs-lookup"><span data-stu-id="68124-137">It is important to understand that the mixed query workload, coupled with concurrency, lead to proper capacity planning for a data warehouse.</span></span>

### <a name="data-management"></a><span data-ttu-id="68124-140">Gestion des données</span><span class="sxs-lookup"><span data-stu-id="68124-140">Data management</span></span>
<span data-ttu-id="68124-141">La gestion des données est essentielle, plus particulièrement si vous prévoyez de manquer d’espace disque dans l’avenir proche.</span><span class="sxs-lookup"><span data-stu-id="68124-141">Managing data is important, especially when you know you might run out of disk space in the near future.</span></span> <span data-ttu-id="68124-142">Les entrepôts de données répartissent généralement les données en plages pertinentes, qui sont stockées en tant que partitions dans une table.</span><span class="sxs-lookup"><span data-stu-id="68124-142">Data warehouses usually divide the data into meaningful ranges, which are stored as partitions in a table.</span></span> <span data-ttu-id="68124-143">Tous les produits basés sur SQL Server prennent en charge l’extraction et l’ajout de partitions dans les tables.</span><span class="sxs-lookup"><span data-stu-id="68124-143">All SQL Server-based products let you move partitions in and out of the table.</span></span> <span data-ttu-id="68124-144">Grâce à ce déplacement de partitions, vous êtes en mesure de déplacer des données plus anciennes vers un modèle de stockage moins coûteux et de conserver les données les plus récentes sur l’espace de stockage en ligne.</span><span class="sxs-lookup"><span data-stu-id="68124-144">This partition switching lets you move older data to less expensive storage and keep the latest data available on online storage.</span></span>

* <span data-ttu-id="68124-148">PolyBase est essentiel à la gestion des données.</span><span class="sxs-lookup"><span data-stu-id="68124-148">PolyBase plays an important role in managing data.</span></span> <span data-ttu-id="68124-149">Si vous recourez à PolyBase, vous avez la possibilité d’archiver des données plus anciennes sur Hadoop ou dans des objets Blob Microsoft Azure.</span><span class="sxs-lookup"><span data-stu-id="68124-149">Using PolyBase, you have the option to archive older data to Hadoop or Azure blob storage.</span></span>  <span data-ttu-id="68124-150">Les données étant dans ce cas conservées en ligne, vous disposez d’un nombre conséquent d’options.</span><span class="sxs-lookup"><span data-stu-id="68124-150">This provides lots of options since the data is still online.</span></span>  <span data-ttu-id="68124-151">La récupération des données de Hadoop peut prendre davantage de temps, mais les économies de stockage réalisées surpassent cet inconvénient.</span><span class="sxs-lookup"><span data-stu-id="68124-151">It might take longer to retrieve data from Hadoop, but the tradeoff of retrieval time might outweigh the storage cost.</span></span>

<span data-ttu-id="68124-153">Un des moyens de rendre les données disponibles pour les rapports et l’analyse est d’envoyer les données conservées sur l’entrepôt de données vers des serveurs dédiés aux rapports et à l’analyse.</span><span class="sxs-lookup"><span data-stu-id="68124-153">One way to make data available for reports and analysis is to send data from the data warehouse to servers dedicated for running reports and analysis.</span></span> <span data-ttu-id="68124-154">Ces serveurs sont appelés mini-Data Warehouses.</span><span class="sxs-lookup"><span data-stu-id="68124-154">These servers are called data marts.</span></span> <span data-ttu-id="68124-155">Par exemple, vous pouvez procéder au pré-traitement des données de rapports, puis les exporter de l’entrepôt de données vers différents serveurs du monde entier afin de les rendre largement disponibles aux clients et aux analystes.</span><span class="sxs-lookup"><span data-stu-id="68124-155">For example, you could pre-process report data and then export it from the data warehouse to many servers around the world, to make it broadly available for customers and analysts.</span></span>
