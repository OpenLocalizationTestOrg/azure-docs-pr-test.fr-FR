## <a name="cluster-support"></a>Prise en charge des clusters
Serveur de sauvegarde Azure peut protéger les données Bonjour suivant des applications en cluster :

-   Serveurs de fichiers

-   SQL Server

-   Hyper-V - si vous protégez un cluster Hyper-V à l’aide de la protection de DPM à grande échelle, vous ne pouvez pas ajouter de protection secondaire pour les charges de travail Hyper-V hello protégé.

    Si vous exécutez Hyper-V sur Windows Server 2008 R2, assurez-vous de mise à jour de hello tooinstall décrite dans l’article [975354](https://support.microsoft.com/en-us/kb/975354).
    Si vous exécutez Hyper-V sur Windows Server 2008 R2 dans une configuration de cluster, veillez à installer SP2 et KB [971394](https://support.microsoft.com/en-us/kb/971394).

-   Exchange Server : le serveur de sauvegarde Azure peut protéger des clusters de disques non partagés pour les versions d’Exchange Server prises en charge (réplication continue en cluster) et peut également protéger Exchange Server configuré pour la réplication continue locale.

-   SQL Server : le serveur de sauvegarde Azure ne prend en charge la sauvegarde des bases de données SQL Server hébergées sur des volumes partagés de cluster.

Serveur de sauvegarde Azure peut protéger les charges de travail de cluster qui sont trouvent dans hello même domaine que le serveur DPM hello et dans un domaine enfant ou approuvé. Si vous souhaitez que les sources de données tooprotect dans des domaines non approuvés ou des groupes de travail, utilisez l’authentification NTLM ou de certificat pour un serveur unique, ou l’authentification par certificat uniquement pour un cluster.
