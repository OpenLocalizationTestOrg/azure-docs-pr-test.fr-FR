<span data-ttu-id="8fe0b-144">Quand vous utilisez des tables de synchronisation, votre code client détermine à quel moment les modifications locales sont synchronisées avec une application principale Azure Mobile App.</span><span class="sxs-lookup"><span data-stu-id="8fe0b-144">When using sync tables, your client code controls when local changes are synchronized with an Azure Mobile App backend.</span></span> <span data-ttu-id="8fe0b-145">Rien n’est envoyé au backend tant que n’a pas été émis un appel pour *envoyer* les modifications locales.</span><span class="sxs-lookup"><span data-stu-id="8fe0b-145">Nothing is sent to the backend until there is a call to *push* local changes.</span></span> <span data-ttu-id="8fe0b-146">De même, le magasin local n’est rempli avec de nouvelles données que si un appel pour *extraire* les données est émis.</span><span class="sxs-lookup"><span data-stu-id="8fe0b-146">Similarly, the local store is populated with new data only when there is a call to *pull* data.</span></span>

* <span data-ttu-id="8fe0b-150">**Extraction**: l’extraction est effectuée sur une table spécifique et peut être personnalisée avec une requête pour récupérer uniquement un sous-ensemble des données du serveur.</span><span class="sxs-lookup"><span data-stu-id="8fe0b-150">**Pull**: Pull is performed on a per-table basis and can be customized with a query to retrieve only a subset of the server data.</span></span> <span data-ttu-id="8fe0b-151">Les Kits de développement logiciel (SDK) clients Azure Mobile insèrent ensuite les données résultantes dans le magasin local.</span><span class="sxs-lookup"><span data-stu-id="8fe0b-151">The Azure Mobile client SDKs then insert the resulting data into the local store.</span></span>

  <span data-ttu-id="8fe0b-163">Si vous souhaitez désactiver la synchronisation incrémentielle, transmettez `null` en tant qu'ID de requête.</span><span class="sxs-lookup"><span data-stu-id="8fe0b-163">If you want to opt out of incremental sync, pass `null` as the query ID.</span></span> <span data-ttu-id="8fe0b-164">Dans ce cas, tous les enregistrements sont extraits à chaque appel à `PullAsync`, ce qui est potentiellement inefficace.</span><span class="sxs-lookup"><span data-stu-id="8fe0b-164">In this case, all records are retrieved on every call to `PullAsync`, which is potentially inefficient.</span></span>
